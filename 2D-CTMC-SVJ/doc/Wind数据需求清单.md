# Wind 数据导出需求清单

**项目**: 2D-CTMC-SVJ 衍生品定价框架
**目的**: 用真实市场数据校准 Heston-SVJ 模型，验证 CTMC 定价方法
**建议合约**: au2412 或 au2502（黄金期货），期权对应同一标的

---

## 1. 期货价格数据（用于估计波动率过程参数）

| 项目 | 说明 |
|------|------|
| 合约 | 黄金期货主力连续合约 |
| 频率 | **日线**即可（5分钟K线更好，但非必须） |
| 字段 | `date, open, high, low, close, volume, open_interest` |
| 时间范围 | **至少6个月**（如 2024-06 ~ 2024-12） |
| Wind命令 | 在Wind终端使用 `WSD` 或Excel插件导出 |
| 导出格式 | CSV |
| 保存位置 | `input/Futures_au2412.csv` |

> 说明：日线足够做波动率参数的初步估计（矩估计/GMM）。如果有5分钟K线，可以做已实现波动率的更精确估计。

---

## 2. 期权行情数据（核心数据，用于校准和验证）

| 项目 | 说明 |
|------|------|
| 标的 | 黄金期货（与期货数据同一标的） |
| 类型 | **看涨 + 看跌**都要 |
| 频率 | **日频**（收盘价/结算价） |
| 字段 | `date, contract_code, option_type, strike, expiry, bid_price, ask_price, settle_price, volume, open_interest` |
| 时间范围 | **至少1个月，覆盖多个到期日** |
| 行权价覆盖 | 需要覆盖 **ATM ± 多个行权价档位**（至少5-7档） |
| 到期日 | 至少 **3个不同到期日**（如1月、2月、3月到期） |
| Wind命令 | `WSD` 导出期权行情 |
| 导出格式 | CSV |
| 保存位置 | `input/Options_au2412.csv` |

> 说明：这是最关键的数据。校准需要多个(K, T)组合的市场价格作为拟合目标。OTM期权对跳跃参数最敏感，ATM期权对波动率参数最敏感。

---

## 3. 无风险利率

| 项目 | 说明 |
|------|------|
| 指标 | **Shibor 3M** 或同期限国债到期收益率 |
| 频率 | 日频 |
| 时间范围 | 与期权数据同期 |
| 导出格式 | CSV |
| 保存位置 | `input/risk_free_rate.csv` |

---

## 4. 合约规格信息

| 项目 | 说明 |
|------|------|
| 内容 | 合约乘数、最小变动价位、行权方式（欧式/美式）、到期日规则、交易时间 |
| 获取方式 | Wind终端 `WSS` 查询或交易所官网 |
| 导出格式 | 随便什么格式都行，能看懂就好 |
| 保存位置 | `input/contract_spec.txt` |

---

## 导出优先级

如果时间有限，按以下顺序导出：

1. **期权日行情**（最重要，校准必须有）
2. **期货日线数据**（次重要，参数估计需要）
3. **无风险利率**（简单，Shibor一条命令）
4. **合约规格**（辅助信息）

---

## 示例 Wind 终端命令

```
# 期货日线
WSD("AU2412.SHF", "open,high,low,close,volume,oi", "2024-06-01", "2024-12-31", "Fill=Previous")

# 期权行情 - 需要找到具体期权合约代码后逐个导出
# 先用 WSET 获取某日所有期权合约列表:
WSET("990002448010000", "2024-11-01")

# Shibor 3M
WSD("M0017133", "close", "2024-06-01", "2024-12-31")
```
